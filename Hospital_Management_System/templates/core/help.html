System Overview

Welcome to the Hospital Management System (HMS). This system manages the end-to-end patient care workflow — from registration through billing.

Typical Patient Flow

  1. Registration — Receptionist registers the patient (or locates an existing record).
  2. Visit — A new visit is created and the patient checks in.
  3. Billing / Payment — The visit invoice is generated; payment is collected or post-pay privilege is approved.
  4. Triage — Nurse/triage officer records vitals; the system auto-calculates priority and clinical notes.
  5. Consultation — Doctor examines the patient, writes notes, and orders labs/imaging/prescriptions.
  6. Diagnostics — Lab and radiology fulfil orders and upload results.
  7. Pharmacy — Pharmacist dispenses prescribed medicines.
  8. Discharge / Referral — Patient is discharged or referred elsewhere.

User Roles

  • Receptionist — Patient registration, visit creation, billing front-desk.
  • Nurse / Triage Officer — Triage vital recording, patient assessment.
  • Doctor — Consultation, diagnosis, ordering tests & prescriptions, editing medical data.
  • Lab Technician — Processing lab requests, entering results.
  • Radiology Technician / Radiologist — Imaging requests & reports.
  • Pharmacist — Dispensing medicines, managing pharmacy stock.
  • Cashier — Processing payments, issuing receipts.
  • System Admin — Full system access, user management, settings.
  • Director — Executive oversight, reports, approvals.
Tip: Use the search bar on any list page to quickly find patients, visits, or records by name, ID, or phone number.

Patients

The Patients module is the master registry for all patients. Every clinical interaction is linked to a patient record.

Registering a New Patient

  1. Navigate to Patients from the sidebar.
  2. Click "Register Patient".
  3. Fill in the required fields: first name, last name, gender, date of birth, phone, address, and next of kin.
  4. Optional fields: email, national ID (NIN), marital status, occupation, nationality, religion, blood group, allergies, chronic conditions.
  5. Click "Create Patient".

Duplicate Detection

As you type the patient's name and date of birth, the system automatically checks for existing records. If a potential duplicate is found, a warning banner appears showing matching patients. Always verify this before creating a new record.

Editing Patient Details

Restricted: Only System Admin and Director can edit patient personal details (name, phone, address, etc.). If you need a correction, contact your administrator or request permission.

Searching for Patients

Use the search bar at the top of the Patients list. You can search by:

  • Patient ID (e.g., KPL-PT-000042)
  • First or last name
  • Phone number
  • National ID / NIN

Patient Detail Page

Click on any patient to see their complete medical history, including triage records, consultations, lab results, imaging, invoices, admissions, and referrals — all in one place.

Visits

A Visit represents a single episode of care. Each time a patient comes to the facility, a new visit is created.

Creating a Visit

  1. Navigate to Visits or open the patient's detail page.
  2. Click "New Visit".
  3. Select the patient (if not already selected).
  4. The visit is automatically assigned a visit number and timestamped.

Visit Statuses

  • Checked In — Patient has arrived.
  • Waiting Triage — Awaiting vitals assessment.
  • Waiting Doctor — Triaged, in queue for consultation.
  • Completed — All care delivered and visit closed.
Note: Visits cannot be edited or deleted by regular staff. Only administrators can manage visit records for data integrity.

Triage

Triage is where patient vitals are recorded and an initial clinical assessment is performed.

Recording Triage

  1. Navigate to Triage.
  2. In the "Eligible Visits" table, find the patient's visit and click "Triage".
  3. Enter all vital signs with their correct metric units:
    • Temperature — in °C (normal: 36.1–37.2)
    • Blood Pressure — in mmHg, format systolic/diastolic (e.g., 120/80)
    • Pulse Rate — in bpm (normal: 60–100)
    • Respiratory Rate — in breaths/min (normal: 12–20)
    • Oxygen Saturation (SpO₂) — in % (normal: 95–100%)
    • Weight — in kg
    • Height — in cm
  4. Describe the patient's symptoms.
  5. Select the outcome: Send to Doctor, Emergency, or Admission.
  6. Click "Save Triage Record".

Auto-Generated Clinical Notes

The system automatically analyses vitals and generates clinical notes when values are abnormal:

  • Fever — Temperature ≥ 38.0°C
  • Hypothermia — Temperature < 35.0°C
  • Hypertension — BP ≥ 140/90 mmHg
  • Hypotension — BP < 90/60 mmHg
  • Tachycardia — Pulse > 100 bpm
  • Bradycardia — Pulse < 60 bpm
  • Hypoxemia — SpO₂ < 95%
  • BMI — Auto-calculated from weight and height

Priority Scoring

Each record gets an automatic priority based on the severity of vital sign abnormalities:

  • Normal — All vitals within range
  • Low Concern — Minor deviations
  • Moderate — Notable abnormalities
  • High — Significant concern
  • Critical — Urgent attention needed

Records are sorted by priority (critical first) so the most urgent patients are seen first.

Editing Triage Data

Doctors, nurses, and triage officers can update the medical vitals on any triage record by clicking the edit button. Patient and visit details are not editable from this form.

Consultation

Doctors use this module to examine patients, document findings, and place orders.

Starting a Consultation

  1. Navigate to Consultation.
  2. Select a patient from the queue (those sent from triage).
  3. The workbench opens showing the patient's vitals, history, and profile.

During Consultation

  • Clinical Notes — Document history of presenting illness, examination findings, and diagnosis.
  • Lab Orders — Request blood work, urinalysis, or other tests.
  • Imaging Orders — Request X-rays, ultrasounds, CT scans, or MRIs.
  • Prescriptions — Prescribe medicines with dosage, frequency, and duration.
  • Referrals — Refer to a specialist or another facility.

Completing a Consultation

After documenting everything, submit the consultation. The system will route orders to the appropriate departments (lab, radiology, pharmacy).

Tip: Review triage auto-notes on the patient's card to quickly spot abnormalities before examining the patient.

Emergency

The Emergency module handles urgent and critical cases that bypass the normal visit workflow.

Registering an Emergency Case

  1. Navigate to Emergency.
  2. Click "New Emergency Case".
  3. Select or register the patient.
  4. Record the nature of the emergency, initial assessment, and vitals.
  5. The case is immediately prioritised for attention.

Emergency Outcomes

  • Stabilised & Discharged — Patient treated and sent home.
  • Admitted — Patient transferred to the admission ward.
  • Referred — Transferred to another facility for specialised care.

Admission

Manages inpatient stays — from bed allocation to discharge.

Admitting a Patient

  1. Navigate to Admission.
  2. Click "Admit Patient".
  3. Select the patient and visit.
  4. Assign a ward, bed, and admitting doctor.
  5. Record the admission diagnosis.

During Admission

  • Track daily rounds and nursing notes.
  • Order additional tests or medications.
  • Monitor patient progress and vitals.

Discharging a Patient

Record the discharge date, outcome, and any follow-up instructions. The system will update the visit status accordingly.

Laboratory

Lab technicians use this module to process test requests from doctors and record results.

Processing Lab Requests

  1. Navigate to Laboratory.
  2. Pending requests appear in the queue.
  3. Click on a request to view patient details and ordered tests.
  4. Perform the test and enter the results.
  5. Submit — the results are automatically available to the requesting doctor.

Test Categories

Tests are organised by category (haematology, biochemistry, urinalysis, etc.). Each test has a defined panel with normal ranges displayed alongside results.

Tip: Flag any critical or panic values immediately by marking them as urgent so the doctor is alerted.

Radiology

Handles all imaging requests — X-rays, ultrasounds, CT scans, and MRIs.

Processing Imaging Requests

  1. Navigate to Radiology.
  2. Pending imaging requests are listed by priority.
  3. Select a request, perform the imaging, and upload the result/report.
  4. The radiologist reviews and adds their interpretation.
  5. Results are sent back to the requesting doctor.

Radiology Reports

Each imaging result includes the technique used, findings, impression, and any uploaded images. These are attached to the patient's medical history.

Pharmacy

The Pharmacy module manages medicine dispensing and stock at the branch level.

Dispensing Medicines

  1. Navigate to Pharmacy.
  2. Pending prescriptions from consultations appear in the queue.
  3. Select a prescription and verify the medicine, dosage, and quantity.
  4. Dispense the medicine — stock is automatically deducted.
  5. A receipt/label can be printed for the patient.

Stock Management

  • View current stock levels for all medicines.
  • Low-stock alerts appear when quantities fall below the reorder threshold.
  • Stock is replenished from the central Medical Store (see Inventory).
Important: Never dispense medicines without a valid prescription from a doctor. All dispensing is logged for audit purposes.

Inventory / Medical Store

The central inventory manages all medical supplies, medicines, and consumables across branches.

Key Features

  • Main Store Dashboard — Overview of all stock items, values, and alerts.
  • Receiving Stock — Record incoming supplies from suppliers with batch numbers and expiry dates.
  • Transfers — Transfer stock from the main store to branch pharmacies, labs, or radiology departments.
  • Pack Sizes — Items can have configurable pack sizes (e.g., a box of 100 tablets). Transfers can be done in packs or individual units.
  • Stock Reconciliation — Physical stock counts can be verified against system records.

Transfer Workflow

  1. Go to Inventory → Transfer.
  2. Select the destination (pharmacy, lab, or radiology).
  3. Choose items and quantities.
  4. Submit — stock is deducted from central and added to the destination.

Billing & Payments

The Billing module handles invoices, payments, and financial tracking for all patient services.

Invoice Lifecycle

  1. Generated — An invoice is auto-created when a visit is initiated or services are ordered.
  2. Pending — Awaiting payment from the patient.
  3. Partial — Some payment received; balance outstanding.
  4. Paid — Full payment received.

Collecting Payment

  1. Navigate to Billing.
  2. Find the invoice (search by patient name or invoice number).
  3. Click "Record Payment".
  4. Enter the amount and payment method (cash, mobile money, card, etc.).
  5. A receipt is generated automatically.

Post-Pay Privilege

In some cases, a cashier can approve a patient to proceed to triage before payment. This requires approval and is logged for audit.

Tip: Use the billing search to find invoices by patient name, invoice number, or status (paid, pending, partial).

Reports

The Reports module provides data-driven insights into hospital operations, finances, and clinical activity.

Available Reports

  • Revenue Summary — Total income by period, branch, and payment method.
  • Patient Statistics — Registration and visit counts over time.
  • Departmental Activity — Lab, radiology, pharmacy throughput.
  • Stock Reports — Inventory levels, transfers, and consumption.
  • Audit Logs — Track who did what and when across the system.

Filtering & Date Ranges

Most reports support date range filters, branch filters, and export options. Use these to narrow down to the exact information you need.

Referrals

Create and track patient referrals to specialists or external facilities.

Creating a Referral

  1. From a consultation or the Referrals module, click "New Referral".
  2. Select the patient and specify the referring doctor.
  3. Enter the referral reason, destination facility, and any clinical notes.
  4. Submit — a referral record is created and linked to the patient's history.

Tracking Referrals

All referrals are listed with their status. Follow up on outcomes and record feedback when available.

Permissions & Access Control

The system uses a two-layer permission model to control who can do what:

Role-Based Access (Layer 1)

Every user has a role (receptionist, doctor, nurse, etc.) which grants baseline access to specific modules. For example, a pharmacist can access Pharmacy but not Laboratory.

Explicit Permission Grants (Layer 2)

Administrators can grant additional module access to specific users beyond their role. For example, giving a nurse access to view billing records.

Requesting Access

  1. If you encounter a "Permission Denied" page, click "Request Access".
  2. Your request is sent to the System Admin / Director for review.
  3. Once approved, you gain access immediately.

Managing Permissions (Admin Only)

  • Navigate to Permissions in the sidebar.
  • View all users and their current access levels.
  • Grant or revoke module access per user.
  • Suspend or reactivate user permissions.
Important: Patient record editing is restricted to System Admin and Director only. Doctors can edit medical/triage data but not patient personal details or visits.

System Settings

Configure the system to match your facility's needs. Only System Admin and Director roles have access.

Branding

  • Upload your facility's logo.
  • Set the facility name displayed throughout the system.
  • Customize colours for the sidebar, dashboard, and buttons.

Branches

If your facility has multiple locations, each is managed as a separate branch. Users are assigned to branches and can only see data from their branch (unless granted multi-branch access).

User Management

  • Create new user accounts and assign roles.
  • Reset passwords for staff members.
  • Deactivate accounts for departed staff.

Billing Configuration

  • Set service prices and fee schedules.
  • Configure payment methods accepted.
  • Set up currency display format.
